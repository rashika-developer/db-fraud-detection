**File**: TRANSACTION_EXAMPLES.md - Section 1 | 5 Examples

```
What: Retail/Point-of-Sale transactions
Risk: Medium (velocity breaches, impossible travel)
Detection: Velocity check on 6+ in 10 min

Examples:
1. Normal daily shopping (Low risk)
2. Department store during travel (Medium risk)
3. Multiple rapid purchases (⚠️ TRIGGERS ALERT)
4. Blacklisted merchant (🚫 BLOCKS)
5. Pattern analysis

SQL Insert:

  Location_City, Latitude, Longitude, Txn_Type)

```

**Fraud Patterns**:
- Velocity breach (6+ txns/10 min)
- Impossible travel (geo-based)
- Merchant category change
- Amount exceeds daily limit
- Blacklist matching

## 🔍 Cross-Type Fraud Scenarios

**File**: TRANSACTION_EXAMPLES.md - Section 5 | sql/05_transaction_examples.sql - Section 5

### Scenario A: Account Takeover
```
Purchase → Withdrawal → Online
Attacker tests all channels in sequence
Detection: Multi-type velocity + risk scoring
```

### Scenario B: "Stripping" Attack
```
Online Purchase → ATM Withdrawal
Attacker verifies card works online, then cashes out
Detection: Immediate withdrawal after online purchase + geospatial
```

### Scenario C: Money Mule Network
```
Incoming Transfer → Multiple Outgoing Transfers
Attacker receives stolen funds, spreads them to multiple accounts
Detection: Rapid multi-account transfers + pattern analysis
```
